Core Infrastructure

The project follows a modular architecture for clear separation of concerns:

  • Agent Identity & Personality: Defines the agent's persona, communication style, and autonomous mindset to act as a Senior Engineer.
  • Engineering Standards Enforcement: Strictly enforces coding standards, security protocols, and scope control to prevent technical debt.
  • Structured Development Workflow: A mandatory state machine (DEFINE → SHIP) that ensures no phase is skipped for non-trivial tasks.
  • Intelligent Skill Routing: Dynamically maps tasks to specialized skillsets like Database Design, Performance Tuning, or API Security.

Specialized Skills

The system activates over 40 specialized skills across the development lifecycle:

Define & Plan

  • Architecting: Planning new cross-cutting features and refactoring large systems.
  • Idea Refinement: Strategic ideation: divergent expansion and convergent refining.
  • Spec-Driven Development: Success criteria before any implementation.
  • Context Engineering: AI-human collaboration and confusion management.

Build & Verify

  • Pattern Implementation: Modern React and .NET development best practices.
  • Source-Driven Development: Verifying patterns against official documentation.
  • Test-Driven Development: Driving feature implementation with failing tests.
  • Security Hardening: Treating input as hostile and protecting against OWASP.

How it Works

Unlike traditional prompts, these configurations act as a System Operating Manual for the AI Agent. Every instruction is built on three pillars:

  • Process over Prose: Workflows are defined as step-by-step executable actions, not just descriptive text. This minimizes "hallucinations" and ensures consistency.
  • Verification Gates: The agent is prohibited from moving to the next phase until specific evidence is provided (e.g., unit tests passing, PRD approved, or security audit completed).
  • Quality Enforcement: Built-in logic that rebuts common excuses for skipping quality steps, forcing the agent to maintain high standards even under pressure.

The system is deployed as a set of global .md and .json files that the agent loads dynamically based on the current context and slash command used.
